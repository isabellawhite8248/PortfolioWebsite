GrassHopper Animation

This was a personal project coded in Java with IntelliJ. I wanted to animate my drawings because I enjoy drawing in my free time and had an idea to utilize digital stipling techniques to compile an array of dots and allow the user to animate any drawing of their choice. The program has two components, one piece allows the user to upload their artwork to the program and trace over it using a pen tool. And another seperate section to animate their drawing. Buttons function to lift the stencil image on and off the page, and a pen tool listens for the drag of the user's mouse, enabling him/her to draw.

Once the drawing is complete, it is then stored in a text file of coordinates. Each coordinate maps a dot location. If the text file is beyond a certain length, the resolution of the image, or how many dot locations is lowered to reduce compile time. After the text file is complete, the artist then compiles the text file into the animation section of the program, where the dots can be manipulated and the image itself can be animated.

In the sample clip, I traced my drawing of a grasshopper and coded bound boxes around each point's original location. Then, at the beginning of the timeline the dots roam aimlessly, but after the shift, signaled by a bottle clinking sound, the dots flood back to their respective bound boxes, stored in an arraylist previously.
